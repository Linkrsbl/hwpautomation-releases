# HWP(한글) 로컬 제어 브릿지 명령어 가이드

이 문서는 ChatGPT, Claude, Antigravity 등 터미널(CLI) 조작이 가능한 AI 에이전트가 사용자의 로컬 환경에 설치된 HWP(한글) 데스크탑 앱을 원격제어하기 위한 명령어 가이드입니다. 

에이전트는 이 문서의 규칙을 완벽하게 숙지하고, 사용자가 요구하는 문서를 자동 작성해야 합니다.

---

## ⚠️ 필수 시작 순서 (이 순서를 반드시 지켜야 합니다)

브릿지가 한글 문서에 접근하려면, 사용자가 **반드시 아래 순서대로** 준비를 완료한 상태여야 합니다.

1. **앱 실행**: HWP 자동화 앱(Electron)을 먼저 실행한다.
2. **한글 문서 열기**: 앱 내에서 작업할 `.hwp` 또는 `.hwpx` 파일을 불러온다.
3. **접근 허용**: 한글 화면에 뜨는 보안 경고 팝업에서 **"모두 허용(N)"** 버튼을 클릭한다.
4. **브릿지 연결 확인**: 앱 UI가 정상적으로 표시되면 브릿지가 한글에 연결된 것이다.
5. **커서 위치 잡기**: 한글 문서에서 **본문 첫 줄을 마우스로 클릭**하여 커서(깜빡이)를 입력 시작 위치에 가져다 놓는다. 브릿지의 `insert` 명령은 현재 커서 위치에서부터 텍스트를 입력하므로, 이 단계를 빠뜨리면 엉뚱한 위치에 입력될 수 있다.
6. **로봇 버튼 복사**: 앱 좌측 메뉴의 🤖 버튼을 눌러 에이전트 프롬프트를 복사한다.
7. **에이전트 실행 및 폴더 열기**: Antigravity 또는 Cursor 등 에이전트를 실행하고, **작업할 한글 문서(.hwpx)가 저장된 폴더**를 프로젝트 폴더로 연다.
8. **프롬프트 붙여넣기**: 복사한 프롬프트를 에이전트 채팅창에 붙여넣으면 에이전트가 가이드를 숙지하고 작업 내용을 요청한다.

> **주의**: 앱과 한글 문서를 켜기 전에 에이전트가 브릿지를 실행하려 하면, 보안 권한이 없어 실패합니다.

## 🆘 문제 발생 시 트러블슈팅 (에이전트 필독)

브릿지 명령어 실행 중 아래와 같은 오류가 발생하거나 한글 문서에 내용이 입력되지 않을 경우:
- `HWP 연결이 없습니다`
- `COM 객체를 찾을 수 없습니다`
- 명령은 성공했는데 문서에 변화가 없음

**사용자에게 즉시 이렇게 안내하라:**

> 🔴 **자동화 준비 순서가 맞지 않아 브릿지 연결에 실패한 것 같습니다.**
> 
> 아래 순서대로 다시 시작해 주세요:
> 1. 현재 열려있는 한글 문서와 앱을 **모두 닫습니다**.
> 2. **앱을 먼저 실행**합니다.
> 3. 앱 내에서 작업할 **한글 문서를 불러옵니다**.
> 4. 한글 화면에 나타나는 보안 경고 팝업에서 **"모두 허용(N)"** 을 클릭합니다.
> 5. 앱 UI가 뜨면, 앱 좌측의 **🤖 로봇 버튼을 다시 눌러** 새 프롬프트를 복사합니다.
> 6. 복사한 새 프롬프트를 저에게 다시 붙여넣어 주시면 이어서 진행합니다.

## 🚀 브릿지 제어 기본 원칙

너는 `HwpBridge.exe`라는 외부 C# 콘솔 프로그램을 호출하여 현재 켜져 있는 한글 문서를 조종할 수 있다. 프로그램의 절대 경로는 사용자가 첫 프롬프트에서 알려줄 것이다. 터미널은 기본적으로 **PowerShell**을 사용한다.

**명령어 호출 형태 (필수 구조)**:
`& "[사용자가 알려준 브릿지 절대 경로]" "HWP_SECURE_TOKEN_2026" --command [명령어] [추가옵션]`
*(주의: 항상 두 번째 인자로 보안 토큰인 `HWP_SECURE_TOKEN_2026`을 반드시 넣어야 프로그램이 작동한다.)*

## 🎯 에이전트 핵심 행동 지침 (필독)

### 1. 템플릿 레벨(수준) 적용을 위한 공백(Indent) 포맷팅
* HWP 앱은 텍스트의 **"앞 공백(스페이스바) 개수"**를 기준으로 문단의 수준(Level)을 판단하고 미리 지정된 템플릿(글꼴, 굵기, 머리 기호 등)을 자동 적용합니다.
* 따라서 너는 텍스트를 출력할 때 `ㅇ 1수준 문장`, `- 2수준 문장` 처럼 리터럴 기호를 직접 입력해서는 **절대 안 됩니다**. 기호가 중복 출력되거나 템플릿이 꼬이는 치명적인 오류가 발생합니다.
* **올바른 방식:**
  오직 **들여쓰기(공백 문자열)**만으로 레벨을 구분하여 순수한 텍스트를 생성해라. (예: 1수준은 공백 0개, 2수준은 공백 2개, 3수준은 공백 4개 등)
```text
[예시 올바른 텍스트 내용]
개요 및 목적 (공백 0개, 1수준 문단)
  본 사업의 목적은 다음과 같다. (공백 2개, 2수준 문단)
    가. 시스템 구축 (공백 4개, 3수준 문단)
```

### 2. 기존 레이아웃 보호 및 제목/내용 타겟팅
* 현재 열려 있는 사업계획서나 문서 파일에는 이미 목차, 겉표지, 고정된 페이지 레이아웃이 설정되어 있습니다.
* 이 고정된 구조는 절대로 건드리지 말고 유지해야 하며, 네가 텍스트를 삽입해야 할 적절한 위치(제목, 본문 내용 영역 등)만 정확하게 타겟팅하여 문서를 완성해라.

### 3. 안정성 (Chunking & 표 분리 작업)
* 한 번에 너무 많은 데이터를 입력하면 시스템 오류나 병목이 발생할 수 있습니다. 한 번 작업 시 **최대 3페이지 분량** 안팎으로 끊어서 순차적으로 텍스트를 입력해라.
* 일반 텍스트 입력과 **표(Table) 입력은 반드시 분리**하여 개별적인 명령어로 요청해라. 텍스트를 먼저 채우고, 표 데이터를 나중에 채우는 등 프로세스를 쪼개어 오류 문제를 최소화해라.

### 4. 간결한 피드백
* 작업을 수행한 후, 사용자에게 장황하게 설명하지 마라.
* "2페이지 본문 작성을 완료했습니다. 다음 단계를 진행할까요?" 수준의 **결과만 간결하고 짧게** 친절히 안내해라.

## 🛠️ 지원되는 명령어 (Commands)

### 1. 텍스트 찾아 바꾸기 (Replace)
문서 내에서 특정 문자열을 찾아 새 내용으로 정확히 교체한다. **지정한 문자열 외 나머지 내용(고정 목차, 제목 등)은 전혀 건드리지 않는다.**

```powershell
& "브릿지경로" "HWP_SECURE_TOKEN_2026" --command replace --target "찾을문자열" --text "새내용"
```

#### ✅ 가장 정확한 사용법 — 사용자가 복사한 텍스트로 타겟팅

> 사용자에게 이렇게 요청하라:  
> *"바꿀 부분의 텍스트를 한글 문서에서 드래그해서 복사한 뒤, 저에게 붙여넣어 주세요."*

사용자가 바꿀 부분을 직접 복사해주면 `--target`에 그 복사된 텍스트를 그대로 사용한다. 이렇게 하면 LLM이 위치를 추측할 필요가 없고, 고정 목차나 제목을 잘못 건드릴 위험도 없다.

#### 📌 여러 줄 블록 교체

단일 줄이 아니라 `ㅇ` 줄과 그 아래 `-` 줄 여러 개를 **한 번에 교체**하고 싶다면, 사용자는 그 **전체 블록**을 드래그해서 복사해 준다. LLM은 그 전체 블록을 하나의 `--target`으로 사용하여 교체한다.

#### ⚠️ 고정 목차는 절대 건드리지 마라

- `4. 관광산업 트렌드 부합성...` 같은 **번호로 시작하는 제목/소제목**은 replace 대상으로 삼으면 안 된다.
- **`ㅇ`(1수준 내용)과 `-`(2수준 내용) 줄**만 교체 대상으로 삼아라.

### 2. 구조화된 데이터 자동 삽입 (Insert)
표 안에 데이터를 채워 넣거나, 특정 양식을 유지하며 방대한 텍스트를 삽입할 때 사용한다. 이 명령어는 JSON 파일을 인자로 받는다.

1. 삽입할 데이터를 분석하여 아래 스키마에 맞는 JSON 데이터 구조를 만든다.
2. 해당 JSON 데이터를 사용자의 PC 임시 경로(예: `C:\temp\payload.json`)에 파일로 저장한다.
3. 아래 명령어를 실행하여 브릿지가 파일을 읽고 문서에 주입하도록 명령한다.

```powershell
& "브릿지경로" "HWP_SECURE_TOKEN_2026" --command insert --file "C:\temp\payload.json"
```

#### [JSON 스키마 — 실제 브릿지 모델 기준]

> ⚠️ **핵심 규칙**: `text` 필드에 `1.`, `ㅇ`, `-` 같은 **머리 기호/번호를 절대 포함하지 마라.**  
> 번호와 기호는 HWP 앱이 `level` 값을 보고 템플릿에서 자동으로 붙여준다.  
> LLM은 오직 `indentSpaces`(공백 개수)와 순수 `text` 내용만 제공하면 된다.

```json
{
  "projectName": "사업계획서 작성",
  "templateId": "[사용자가 알려준 템플릿 이름]",
  "sections": [
    {
      "id": "section-1",
      "paragraphs": [
        {
          "id": "p1",
          "level": 1,
          "indentSpaces": 0,
          "text": "시장 분석 및 필요성",
          "isBold": false,
          "fontSize": 0,
          "lineSpacing": 0
        },
        {
          "id": "p2",
          "level": 2,
          "indentSpaces": 2,
          "text": "국내 시장 규모는 지속적으로 성장하고 있으며 연평균 성장률은 약 12%에 달한다.",
          "isBold": false,
          "fontSize": 0,
          "lineSpacing": 0
        },
        {
          "id": "p3",
          "level": 3,
          "indentSpaces": 4,
          "text": "특히 중소기업 대상 솔루션 수요가 급증하고 있다.",
          "isBold": false,
          "fontSize": 0,
          "lineSpacing": 0
        }
      ]
    }
  ]
}
```

**필드 설명:**
| 필드 | 타입 | 설명 |
|---|---|---|
| `level` | int | 문단 수준 (1=대항목, 2=중항목, 3=소항목) |
| `indentSpaces` | int | 앞 공백 개수. 수준을 결정하는 핵심 값 |
| `text` | string | **순수 텍스트만**. 기호/번호 포함 금지 |
| `isBold` | bool | 굵게 여부 (0=일반, 기본은 템플릿 따름) |
| `fontSize` | int | 글자 크기 (0이면 템플릿 기본값 사용) |
| `lineSpacing` | int | 줄간격 (0이면 템플릿 기본값 사용) |
